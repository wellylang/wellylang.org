• Goal: combine the advantages of C, Python and Java.
  • C is fast and powerful, but too low-level.
  • Java is fast and high-level, but you're stuck in a "walled garden”.
  • Python is powerful and high-level, but will never be fast.

In a given project, it is often necessary to choose between these languages (or similar). Choice has to be made early, and can have a big effect on the success of the project. For example:

  • Choose C and you will always find complex things difficult.
  • Choose Python and you will always have limited performance.
  • Choose Java and you will always be in a "walled garden”.

There is room for a language between these.

Sub-goals

  • Simplify the tool chain. Who needs a "compiler" tool? Only the computer should care.
  • Provide proper structs. Especially, provide arrays of structs, and functions to/from structs.
  • Provide proper (tagged) unions. If Java needed enumerations (it did), it also needs unions.
  • Unify the language with its own preprocessor language. If they're different, at least one must be wrong.
  • Don't make any basic mistakes (see later).
